# UI Building Guidelines

## Purpose

Build modern, responsive, accessible interfaces.

---

# Design Principles

Always prioritize:

- consistency
- simplicity
- readability
- accessibility
- responsiveness

---

# Design Tokens

Never use arbitrary colors.

Use project design tokens.

Example:

- zinc
- slate
- emerald
- blue
- red
- amber

Support dark mode using `dark:` variants.

---

# Responsive Design

Design mobile-first.

Support:

- Mobile
- Tablet
- Desktop

Avoid horizontal scrolling.

---

# Layout

Use:

- flex
- grid
- gap utilities

Avoid:

- fixed heights
- fixed widths

Prefer:

- h-full
- min-h
- flex-1

---

# Components

Maintain consistent:

- spacing
- border radius
- shadows
- typography
- colors

---

# Accessibility

Always:

- use semantic HTML
- maintain sufficient contrast
- support keyboard navigation
- provide focus states
- use accessible labels

---

# Visual Hierarchy

Use hierarchy through:

- spacing
- typography
- size
- weight

Never rely solely on color.
