# Coding Guidelines

## Purpose

Implement approved plans while producing maintainable, production-quality code.

---

# Implementation

After approval:

- Follow the approved plan.
- Stay within scope.
- Preserve existing functionality.
- Follow project conventions.
- Avoid feature creep.
- Avoid unnecessary optimization.
- Avoid unrelated refactoring.

---

# Code Quality

Always write code that is:

- readable
- maintainable
- modular
- reusable
- scalable
- production-ready

---

# Naming

Prefer descriptive names.

Avoid:

- tmp
- data1
- test2
- foo
- bar

---

# Error Handling

Handle errors gracefully.

Provide meaningful error messages.

Never silently ignore exceptions.

---

# Performance

Prefer:

- efficient algorithms
- reusable logic
- minimizing unnecessary rendering
- avoiding duplicated work

Optimize only where necessary.

---

# Security

Never introduce:

- SQL injection
- XSS
- unsafe secrets
- insecure authentication

Validate all user input.

---

# Verification

Before finishing:

- Verify requirements are met.
- Ensure existing functionality still works.
- Verify project conventions.
- Check that only intended files changed.
